# Sector dossiers — generated gather-by-reference views

**Binding from 2026-06-09.** Operator-authorised folder-structure decision: keep
the orthogonal top-level trees; add a generated per-sector dossier instead of
physically nesting `claims/` under `theory/sector-X`.

## 1. Decision

`claims/` stays flat — claim IDs already encode the sector (`A1`, `B1`, `C5`, …).
Physically nesting claims under `theory/sector-X/` was rejected: it would break
hundreds of cross-reference paths, require a large operator-side `git mv` (the
sandbox mount blocks moves), and conflate verification packages (`claims/`) with
narrative (`theory/`), predictions, and the failure ledger — which are orthogonal
governance axes. The per-sector "one store" experience is instead delivered as a
GENERATED view that gathers the orthogonal trees by reference.

## 2. Output

```
theory/sectors/INDEX.md   one-screen TOE scorecard (per sector: claim count,
                          tier profile, open-gate count, prediction count)
theory/sectors/<X>.md     per sector A–F: synthesis link + claims (tier /
                          named hypotheses / open gates) + predictions +
                          negative results — all by reference
```

Generated by `verification/scripts/build_dossier.py`. Never hand-edit.

## 3. Sources of truth (unchanged)

```
claims/<ID>/status.json           sector, tier, hypotheses, open gates, title
theory/sector-<X>-*/README.md     hand-written synthesis (linked, not copied)
predictions/prediction-ledger.md  predictions (claim-link -> sector)
negative-results/registry.md      negatives (claim-prefix -> sector)
```

The dossier copies nothing load-bearing; it links. Editing a claim card and
re-running the generator refreshes every dossier.

## 4. Enforcement

`release_check.py` runs `build_dossier.py --check` (the `[dossier]` gate): the
files must equal the generator output, exactly as CLAIMS / CATALOG / TODO /
CHANGELOG are sync-gated. The catalog includes `theory/sectors/*.md`.

## 5. Entry points

Top-down: [`claims/INDEX.md`](../claims/INDEX.md) (all sectors, one screen) and
per-claim `claims/<ID>/INDEX.md`. Per-sector:
[`theory/sectors/INDEX.md`](../theory/sectors/INDEX.md) -> `theory/sectors/<X>.md`.
Both are generated and never drift from the claim cards.
