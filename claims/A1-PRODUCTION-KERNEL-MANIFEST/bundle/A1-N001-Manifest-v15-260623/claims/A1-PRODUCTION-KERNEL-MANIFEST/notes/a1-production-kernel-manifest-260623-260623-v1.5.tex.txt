\section*{1. Purpose and scope}

Certifying the scalar Brazovskii slice as the real N-001 kernel requires that the
verifier evaluate the ORIGINAL \texttt{bloch\_matrix\_linear} under the EXACT runtime
settings, not checker-chosen defaults. The function's scalar diagonal depends on
\texttt{laplacian\_mode}, \texttt{bcc\_mix\_epsilon}, \texttt{a\_bcc},
\texttt{family\_masses}, \texttt{k\_lock} (function DEFAULT $0.15$) and
\texttt{eta\_shell}; v1.4 hardcoded two of these. v1.5 reads all of them from the
config and passes them verbatim. Verification:
\url{codes/foundations/a1_kernel_checks.py} v1.5 (14/14 PASS); solver sources
\url{codes/foundations/n001_solver/}; config
\url{claims/A1-PRODUCTION-KERNEL-MANIFEST/canonical_n001_kernel.json}.

The config stores the full param set passed to the original function:
\begin{equation}
\begin{aligned}
\texttt{scalar\_slice}=\{&\texttt{laplacian\_mode}{=}\text{spectral},\ \texttt{bcc\_mix\_epsilon}{=}0,\ \texttt{a\_bcc}{=}1,\\
&\texttt{family\_masses}{=}\mathbf{0},\ \texttt{k\_lock}{=}0,\ \texttt{eta\_shell}{=}0\}.
\end{aligned}
\end{equation}
Under these, the ORIGINAL $\texttt{bloch\_matrix\_linear}(\{|k|,0,0\},\{r,Z,Y,q_0,
\texttt{**scalar\_slice}\})_{00}$ equals the stored $r_{\rm zero}+Z|k|^2+Y|k|^4$ to
$8.9\times10^{-16}$. Each setting is load-bearing (reverting it shifts the ORIGINAL
diagonal): $\texttt{k\_lock}{=}0.15$ (the function default) $\to0.100$ via
$k_{\rm lock}(I-P_0)_{00}=0.15\cdot\tfrac{2}{3}$; $\texttt{eta\_shell}{=}0.1\to0.231$;
$\texttt{mixed\_bcc}(\epsilon{=}0.5)\to1.997$; $\texttt{family\_masses}\to0.050$.
Thus fixing them in the CONFIG (read by the checker) rather than in the checker is
essential to verifying the runtime symbol.
