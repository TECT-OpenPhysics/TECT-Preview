\section*{5. Devil's-advocate (three objections; CLAUDE.md 6.3)}

\begin{itemize}
\item[($\alpha$)] \emph{``An unexecuted script can contain trivial
bugs (typos, API drift) that make this unit's value illusory.''}
VALID-with-mitigation: certain --- which is why the script carries
\texttt{\_\_status\_\_ = DRAFT - NOT YET EXECUTED} in its module
docstring AND this note's banner, the canary assert (S4) isolates
environment breakage from mathematical failure, and the first-run
protocol is written into section 3. The alternative (leaving eleven
checks as prose registrations) is strictly worse for the next
session.
\item[($\beta$)] \emph{``Encoding estimate-grade numbers as asserts
with tolerances risks circularity: the script `verifies' what the
notes guessed.''} DISMISSED with the design: the asserts' EXPECTED
values are the notes' arithmetic, but the ACTUAL values come from the
production helpers (\texttt{M\_fast}, the $J$ quadrature) --- an
independent computation path; agreement is verification, disagreement
fires the notes' own pre-registered gates. The dressed-ratio
tolerances (15\%) are wide precisely because those numbers are
estimates pending M-ENDPOINT.
\item[($\gamma$)] \emph{``The script writes its artefact under
B1/runs although it covers B2 and B5 checks too --- claim-card
hygiene.''} VALID-with-mitigation: the single-artefact choice keeps
the first run atomic; on full PASS the per-claim JSON split (B2's S1,
B5's S3/S4/S7 slices) is a five-line follow-up, registered in the
first-run protocol. The \texttt{\_\_claims\_\_} header lists all
three claims explicitly.
\end{itemize}
